Feat Cloud 提供了一种面向企业级应用开发的解决方案，它的定位有点像 SpringBoot。

**设计方面**：Feat Cloud 尽最大可能保留了 SpringBoot 的使用习惯，以此降低开发者的学习成本。

**技术方面**：Feat Cloud 在编译期对代码进行静态分析，最大化提升服务运行性能，降低资源消耗。

## 准备工作

### 引入 Maven 依赖
创建一个 Maven 项目，在 `pom.xml` 文件中添加以下依赖：
```xml title=pom.xml
<dependency>
    <groupId>tech.smartboot.feat</groupId>
    <artifactId>feat-cloud-starter</artifactId>
    <version>${feat.version}</version>
</dependency>
```
<Aside>
    **$\{feat.version}**
    需根据实际情况设置，建议使用[最新版本](https://central.sonatype.com/artifact/tech.smartboot.feat/feat-cloud-starter)。
</Aside>

### 配置 IDEA
调整 IDEA 编译配置，以启用 Feat Cloud 的静态优化功能，**否则请求将无法正常路由**。

设置路径为：`Preferences -> Build, Execution, Deployment -> Build Tools -> Maven -> Runner`，
勾选`Delegate IDE build/run actions to Maven` 并点击 `OK` 保存配置。

## 快速启动
在 Maven 工程中创建一个 `FeatCloudDemo.java` 文件，添加以下代码：
``` java title=FeatCloudDemo.java
@Controller
public class FeatCloudDemo {
    @RequestMapping("/cloud")
    public String helloWorld() {
        return "hello Feat Cloud";
    }

    public static void main(String[] args) {
        FeatCloud.cloudServer().listen();
    }
}
```
启动程序，打开浏览器访问 `http://localhost:8080/cloud`。

## 核心特性

Feat Cloud 是基于注解的方式进行服务开发，与传统框架在运行时解析注解不同，**Feat Cloud 在编译时对代码进行静态分析**，实现了以下核心优势：

### 🚀 零反射架构
- 运用 APT（Annotation Processing Tool）技术
- 编译期生成优化代码，消除运行时反射开销
- 结合 ServiceLoader 实现零配置服务加载

### ⚡ 极致性能
- 启动时间 < 100ms
- 极低内存占用
- 高并发处理能力

### 🔧 开发友好
- 保持 SpringBoot 使用习惯
- 支持 AOT 和 AOT VM 双模式
- 完善的依赖注入和生命周期管理

<Aside type="tip">
**深入了解技术原理**：想要了解 Feat Cloud 的 AOT 编译机制、代码生成过程等技术细节，请参考 [AOT 编译和 AOT 虚拟机](/feat/cloud/aot/) 章节。
</Aside>

## 快速上手指南

### 运行模式选择

Feat Cloud 提供两种运行模式：

- **AOT 模式**：适用于生产环境，极致性能，零反射
- **AOT VM 模式**：适用于开发环境，保持热重载能力

默认情况下，框架会自动选择合适的模式。开发阶段建议使用 AOT VM 模式以提高开发效率。

## Bean 生命周期管理

Feat Cloud 中对于托管的实例的生命周期管理分为：**创建** -> **属性填充** -> **初始化** -> **使用** -> **销毁** 几个核心阶段。

<img src={lifecycle.src} alt="Feat Cloud 生命周期" width="100%" className="shadow"/>

### 核心注解

#### @Bean / @Controller
用于定义需要托管的实例对象。

```java
@Bean
public class UserService {
    // 业务逻辑
}

@Controller
public class UserController {
    @RequestMapping("/users")
    public String getUsers() {
        return "users";
    }
}
```

#### @Autowired
用于依赖注入，支持循环依赖。

```java
@Bean
public class UserController {
    @Autowired
    private UserService userService;  // 自动注入
    
    // setter 方法（可选）
    public void setUserService(UserService userService) {
        this.userService = userService;
    }
}
```

#### @PostConstruct / @PreDestroy
用于生命周期管理。

```java
@Bean
public class DatabaseService {
    @PostConstruct
    public void init() {
        // 初始化数据库连接
    }
    
    @PreDestroy
    public void destroy() {
        // 释放资源
    }
}
```

<Aside type="note">
**注意**：所有 Bean 默认都是单例的，在整个应用中只有一个实例。
</Aside>
